# CatScope v0.6.3-preview

macOS Universal Preview preparation for CatScope, a lightweight Android Logcat workbench for Windows and macOS preview users.

## Highlights

- Added a macOS universal preview build path for both Intel Macs and Apple Silicon Macs.
- Added DMG packaging with a SHA256 checksum file for GitHub Release assets.
- Added stable macOS bundle identifiers: `com.hbladeh.catscope` for release builds and `com.hbladeh.catscope.dev` for dev builds.
- Updated setup and QA documentation for macOS preview usage and adb discovery.

## Requirements

- macOS with Wails-supported WebKit runtime, or Windows with Microsoft WebView2 Runtime for Windows builds.
- Android SDK Platform Tools with `adb` available through `PATH`, `ANDROID_HOME`, `ANDROID_SDK_ROOT`, or CatScope configuration.
- USB debugging enabled and authorized for live device logging.
- An Android device or emulator for live Logcat; offline log and session workflows can be used without a connected device.

## macOS Installation

1. Download `CatScope-v0.6.3-preview-macos-universal.dmg` from the GitHub Release assets.
2. Optionally verify `CatScope-v0.6.3-preview-macos-universal.dmg.sha256`.
3. Open the DMG and drag `CatScope.app` to Applications.
4. Start CatScope, select an Android device and click **Start Logcat**, or open an offline log/session file.

This preview DMG is self-signed/ad-hoc signed and is not Apple-notarized yet. If macOS Gatekeeper blocks the first launch, use Finder's **Open** action from the context menu or allow the app in System Settings. A later formal release should use Apple Developer ID signing, notarization, and stapling.

## Known Issues

- CatScope is not a complete IDE and is not an Android Studio replacement.
- macOS support is preview-level. Please report adb discovery, device connection, file dialog, and Gatekeeper issues with OS version and hardware architecture.
- GUI apps launched from Finder may not inherit the same shell `PATH`; if adb is not detected, configure the full adb path such as `/Users/<you>/Library/Android/sdk/platform-tools/adb`.
- Gradle Project Sync and code editing are not supported.
- Module and variant selection in Build / Install / Launch is still basic.
- External AI APIs are not integrated; AI Context is local Markdown generation only.
- Vite may show a chunk size warning during build; this does not currently block release preparation.
