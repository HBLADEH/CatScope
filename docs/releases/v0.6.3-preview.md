# CatScope v0.6.3-preview

这个版本主要补齐 macOS universal 预览包，让 CatScope 可以在 Windows 之外，也先跑在 Intel Mac 和 Apple Silicon Mac 上。

## 重点变化

- 新增 macOS universal 构建流程，一个包同时支持 Intel Mac 和 Apple Silicon Mac。
- 新增 DMG 打包和 SHA256 校验文件，方便作为 GitHub Release 资产发布。
- 固定 macOS bundle id：正式构建使用 `com.hbladeh.catscope`，开发构建使用 `com.hbladeh.catscope.dev`。
- 修复 macOS 手动选择 adb 时文件选择框崩溃的问题。
- 更新 macOS 安装、adb 路径、QA 和发布资产说明。

## 运行要求

- macOS，或 Windows + Microsoft WebView2 Runtime。
- Android SDK Platform Tools，并确保 `adb` 能通过 `PATH`、`ANDROID_HOME`、`ANDROID_SDK_ROOT` 或 CatScope 配置找到。
- 如果要看实时 Logcat，需要设备开启 USB 调试，并允许当前电脑授权。
- 离线日志和 session 查看不依赖连接设备。

## macOS 安装方式

1. 从 GitHub Release 下载 `CatScope-v0.6.3-preview-macos-universal.dmg`。
2. 如需校验完整性，可以使用同名 `.sha256` 文件。
3. 打开 DMG，把 `CatScope.app` 拖到 Applications。
4. 启动 CatScope，选择 Android 设备并点击 **Start Logcat**；也可以直接打开离线日志或 session 文件。

这个预览版 DMG 目前是 self-signed/ad-hoc signed，还没有做 Apple notarization。第一次启动如果被 macOS Gatekeeper 拦截，可以在 Finder 里右键选择 **Open**，或到系统设置里允许该应用。后续正式面向普通用户发布时，建议接入 Developer ID 签名、notarization 和 stapling。

## 已知限制

- CatScope 不是完整 IDE，也不是 Android Studio 替代品。
- macOS 支持仍是 preview 级别。遇到 adb 检测、设备连接、文件选择框、Gatekeeper 相关问题时，请带上 macOS 版本和芯片架构一起反馈。
- 从 Finder 启动的 GUI 应用不一定继承 shell 里的 `PATH`。如果 CatScope 找不到 adb，请手动配置完整路径，例如 `/Users/<you>/Library/Android/sdk/platform-tools/adb`。
- Build / Install / Launch 目前还是基础能力，不做 Gradle Project Sync，也不包含代码编辑器。
- AI Context 只在本地生成 Markdown，不调用外部 AI API。
- 前端构建时 Vite 可能提示 chunk size warning，目前不影响发版。
