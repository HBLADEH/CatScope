# CatScope v0.6.2-preview

First GitHub Preview Release preparation for CatScope, a lightweight Android Logcat workbench for Windows.

## Highlights

- Lightweight Android Logcat Workbench for focused Android troubleshooting without opening Android Studio.
- Package-aware filtering with package selection, PID tracking, level filters, tags, keywords, exclusions, and regex search.
- Rule-based Crash Analyzer for Java crashes, ANR, native crashes, JNI errors, and install failures.
- Local AI Context Generator that creates Markdown context without calling external AI APIs.
- Offline Log Viewer and `.catscope-session` restore flow for returning to a saved debugging state.
- README branding, final logo usage, screenshots, demo recording guide, and release asset checklist are ready for the first public preview.

## Features

- Device discovery through local adb and Android SDK Platform Tools.
- Live `threadtime` Logcat streaming with a large ring buffer and virtualized frontend table.
- Search, level, tag, exclude keyword, regex, package, and PID tracking filters.
- Offline `.txt`, `.log`, `.jsonl`, and CatScope session workflows.
- TXT / JSONL export and local Markdown AI context export.
- Basic Build / Install / Launch workflow for Gradle wrapper projects.
- Workspace and filter preset persistence in local CatScope configuration.
- Screenshot documentation for Live Logcat, Analysis, and Build / Install / Launch views.

## Requirements

- Windows with Microsoft WebView2 Runtime installed.
- Android SDK Platform Tools with `adb` available through `PATH`, `ANDROID_HOME`, `ANDROID_SDK_ROOT`, or CatScope configuration.
- USB debugging enabled and authorized for live device logging.
- An Android device or emulator for live Logcat; offline log and session workflows can be used without a connected device.

## Installation

1. Download `CatScope.exe` from the GitHub Release assets.
2. Optionally verify the SHA256 checksum below.
3. Start `CatScope.exe`.
4. Select an Android device and click **Start Logcat**, or open an offline log/session file.

## Known Issues

- CatScope is not a complete IDE and is not an Android Studio replacement.
- Gradle Project Sync and code editing are not supported.
- Module and variant selection in Build / Install / Launch is still basic.
- Launcher activity automatic parsing is not complete.
- External AI APIs are not integrated; AI Context is local Markdown generation only.
- Large session files are not compressed or split.
- Vite may show a chunk size warning during build; this does not currently block release preparation.
- `Logo.candidate.png` is a raw logo candidate with a white background and should not be used in README, release assets, app icons, or screenshots.
