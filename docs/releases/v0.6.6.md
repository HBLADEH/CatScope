# CatScope v0.6.6

这个版本为 CatScope 加入自动检测升级，并让 Windows 便携版在保持单 EXE 分发的同时支持校验后安装和重启。发版流程也增加了版本一致性保护。

## 新增

- 启动后自动检查 GitHub Release，也可以在左侧“应用更新”区域手动检查。
- 默认只接收正式版，可选择接收 Preview 版本。
- Windows 便携版支持“安装并重启”：下载当前架构对应的 EXE，校验同名 SHA256，退出后替换原文件并重新启动。
- 自动替换不可用时，可以直接打开对应 GitHub Release 手动下载。

## 改进

- 程序运行时版本、Windows 资源版本、前端包版本统一为 `0.6.6`。
- Release workflow 会在 Windows 和 macOS 构建前校验源码版本与 tag 是否一致，不一致时停止发版。
- Windows Release 继续同时提供版本化 EXE 和 `.sha256` 文件，供用户校验和应用内升级使用。
- 前端生产构建会先清理旧的 dist 资源，避免历史 JS / CSS 被重复打进 EXE。

## 使用方式

1. 启动 CatScope，默认会在约 1.5 秒后静默检查一次正式版更新。
2. 在左侧“应用更新”区域可以关闭自动检查、接收 Preview，或手动检查。
3. Windows 发现新版本后点击“安装并重启”。下载和 SHA256 校验完成后，CatScope 会自动退出、替换并重新打开。
4. 如果 EXE 所在目录不可写，请点击“查看 Release”并手动下载替换。

## 已知限制

- macOS 当前支持检测更新和打开 Release 页面，但不会自动替换 `.app`；后续应结合 Developer ID 签名和 notarization 接入。
- Windows EXE 如果位于无写权限目录，应用内替换会失败。
- 当前完整性校验使用 GitHub Release HTTPS 和 SHA256。面向更广泛用户发布时，仍建议为 Windows EXE 增加 Authenticode 数字签名。

## 运行要求

- Windows 10 / 11 + Microsoft WebView2 Runtime，或 macOS。
- Android SDK Platform Tools；实时 Logcat 仍需要已授权的 Android 设备或模拟器。

## 校验方式

- 运行 `go test ./...`。
- 在 `frontend/` 运行 `npm run build`。
- 运行 `go run scripts/check-version.go v0.6.6`。
- Windows 使用 `scripts/check.ps1` 和 `scripts/build-windows.ps1`。
- Release 中 Windows EXE 和 macOS DMG 均提供同名 `.sha256` 文件。
