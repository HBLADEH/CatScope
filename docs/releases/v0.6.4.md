# CatScope v0.6.4

这个版本主要改善 Logcat 搜索和日志表格阅读体验，同时修复发布 workflow 里前端依赖没有安装就开始构建的问题。

## 重点变化

- 搜索框支持字段查询语法，例如 `tag:ActivityManager`、`pid:1234`、`level:E`、`message:error`、`-message:noise`。
- 字段查询支持包含、精确匹配、前缀匹配和正则匹配，便于在大量日志里快速缩小范围。
- 普通关键词搜索会同时覆盖 package、PID、TID、level、tag、message 和 raw 文本。
- 清空搜索时会同步重置 level、tag、排除词和 regex 状态，回到更直观的默认过滤条件。
- 日志表格改为按实际行高测量虚拟滚动项，长 message 最多显示 4 行，减少内容被截断后难以判断的问题。
- Release workflow 会在 Windows 和 macOS 构建前执行 `npm ci --prefix frontend`，避免 CI 找不到 `vue-tsc`。
- 协作说明补充规则：如果为了测试启动了前端开发服务，测试结束后要主动关闭。

## 运行要求

- Windows + Microsoft WebView2 Runtime，或 macOS。
- Android SDK Platform Tools，并确保 `adb` 能通过 `PATH`、`ANDROID_HOME`、`ANDROID_SDK_ROOT` 或 CatScope 配置找到。
- 如果要看实时 Logcat，需要设备开启 USB 调试，并允许当前电脑授权。
- 离线日志和 session 查看不依赖连接设备。

## 使用提示

1. 在搜索框输入普通关键词，可以继续按文本搜索日志。
2. 输入 `tag:ActivityManager` 可以只看 tag 包含 `ActivityManager` 的日志。
3. 输入 `pid:1234 level:E` 可以同时限定 PID 和日志级别。
4. 输入 `-message:noise` 可以排除 message 中包含 `noise` 的日志。
5. 长日志 message 会在列表里显示多行预览，点击日志行仍可在详情面板查看完整内容。

## 已知限制

- CatScope 不是完整 IDE，也不是 Android Studio 替代品。
- Build / Install / Launch 目前还是基础能力，不做 Gradle Project Sync，也不包含代码编辑器。
- AI Context 只在本地生成 Markdown，不调用外部 AI API。
- 前端构建时 Vite 可能提示 chunk size warning，目前不影响发版。
