## Maintenance

- When user-visible behavior changes, update the README, user guide, and relevant QA items.
- When module boundaries change, update the architecture document.
- When release flow or artifacts change, update Release Assets, the Release Notes template, and the QA checklist.
- Internal exploration material, such as MVP Tasks and Codex Start Prompt, is not a primary external-user document; label its intended audience when maintaining it.
